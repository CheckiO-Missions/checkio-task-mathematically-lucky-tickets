Концепція "Математично щасливих квитків" схожа на ідею російських "щасливих квитків". Вона посилається до старих квитків громадського транспорту з надрукованими на них числами із 6 цифр.

Тобі дається номер квитка, і поєднання його цифр може стати математично щасливим, дотримуючись цих правил: 

    1. Цифри числа можна розділити на групи.
    2. Ти не можеш змінити порядок груп чи цифр.
    3. Кожна група трактується як одне ціле число (1 і 2 стає 12 і т.д.).
    4. Знаки дій (+, -, * та /) застосовуються між групами.
    5. Дужки розміщуються навколо під-виразів, щоб усунути будь-яку неоднозначність у порядку дій.

Наприклад: 

   
    * 238756 -> (2 * (38 + ((7 + 5) * 6)))
    * 000859 -> (0 + (00 + (8 + (5 + 9))))
    * 561403 -> (5 * (6 + (1 + (40 / 3))))

Квиток вважається математично щасливим, якщо немає такої комбінації його цифр, знаків дій і порядку дій, що дадуть в результаті виконання 100. Наприклад: 

    
    * 000000 очевидно щасливе, адже незалежно від того, яке поєднання ти вибудуєш,
    результат все одно буде 0;
    * 707409 і 561709 також щасливі - з них ніяк не отримати 100;
    * 595347 нещасливе: (5 + ((9 * 5) + (3 + 47))) = 100;
    * 593347 нещасливе: (5 + ((9 / (3 / 34)) - 7)) = 100;
    * 271353 нещасливе: (2 - (7 * (((1 / 3) - 5) * 3))) = 100.

Результат виконання комбінації повинен бути рівно 100, щоб число вважалося нещасливим. Дроби можуть траплятися в проміжних розрахунках (як, наприклад, у наведених вище прикладах для 593347 та 271353), але результат повинен бути цілим числом.

Завдання: Маючи 6-значний номер квитка, твоя функція повинна визначити, чи є він математично щасливим.

Вхідні дані: 6-значне число як рядок.

Вихідні дані: Воно математично щасливе чи ні, як логічне (булеве) значення.
